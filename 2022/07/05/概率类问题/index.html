HDU 4870

思路

感觉利用关系写出式子

首先看只有一个账号的情况。

设dp i 是 i -> i+1 需要的期望

dp[ i ] = 1*p + （ 1 - p ）* ( dp[ i - 2 ] + dp [ i -1 ] + dp [ i ] + 1 )

成功+失败(浪费一次机会+3次成功)

其中01初始化要注意

考虑两个账号时

两个账号本质是独立的，因为最后的结果必然是 950 1000

所以只需要 2sum - dp [ 19 ]

CF 148D

https://codeforces.com/problemset/problem/148/D

思路

一眼概率DP

设状态(i,j)为i个白球、j个黑球，公主能赢的概率

由以下三种情况累加

如果公主拿到白球

i/(i+j)

如果公主黑球、龙黑球、消失了白球 (i>=1&&j>=2)

j/(i+j) * (j-1)/(i+j-1) * (i/i+j-2)

如果公主黑球、龙黑球、消失了黑球(j>=3)

j/(i+j) * (j-1)/(i+j-1) * (j-2)/(i+j-2)

处理边界情况

LightOj 1408

https://lightoj.com/problem/batting-practice

思路

题解说不要纠结于当前是第几轮，值得感叹的是我一开始就没想过这事。

题解

确定状态fi为连续i次成功距离结束的期望

状态gi为连续i次失败距离结束的期望

f[i]=p∗f[i+1]+(1−p)∗g[1]+1

g[i]=(1−p)∗g[i+1]+p∗f[1]+1

令i=1，递推出f1和g1的结果

一般求概率是正推，求期望是逆推。通过题目可以体会到这点。

还是根据状态写方程，化简即可。

概率类问题和操作次数虽然有关但是不可能以操作数列方程，根据状态来才比较好

ZOJ 3329

ZOJ (pintia.cn)

成环的概率

思路

首先是找好状态

三个筛子的结果多样而且不同结果有不同的分数，并且最后的胜利以分数论，所以状态应该选用分数

dpi=当前分数为i时所需期望

dpi= r dp0 + 求和 px * dpx+i +1

当 x+i >=n dpx+i=0

r=1/(k1*k2*k3)

出现了dp0，有环

用换元的思想

把dp0看成常数，

dpi=Ai * dp0 + Bi

带入

dpi=(求和pk*A[i+k]+p0)dp[0]+求和pk*B[i+k]+1

Ai=(求和pk*A[i+k]+p0)

Bi=(求和pk*B[i+k])

dp0 = B[0] /(1-A[0])
