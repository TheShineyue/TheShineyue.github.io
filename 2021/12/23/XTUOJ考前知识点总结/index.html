XTUc语言期末考注意事项

发表于2021-12-23更新于2021-12-23

略述期末考常考的一些知识点和考试的时候要注意的东西

注意

  1. 考试一般前5道题是大家都能做的，无非就是用时长短的问题。学的好的同学要把第六题第七题也视作自己的目标去解决，最后的第八题看运气了。。。（反正我当年没写出来去年的第八题）
  2. 2题左右能及格，6题左右满绩不是问题
  3. 心态很重要！ 平时周四下午或周末可以规定一段时间给自己当模拟测试，熟悉acm式的期末考。此外，也要相信题目肯定在自己的能力范围内，冷静思考疏漏之处。
  4. 罚时是 你交这道题的用时+错误次数*20分钟，所以有的时候要多想想，”真的考虑到所有细节了吗？“再交，不然交太多次罚时爆炸，让你拿不到应该得到的分数。
  5. 考试流程：签到题稳稳出（前两题要考虑到所有细节），中等题遇到不会先跳（因为有的时候题目会有点坑，去年的第五题好像就是出了一点问题，我因为看不懂跳过写下题而节约了不少罚时），做到第六题回过头解决刚刚跳过的题目，最后解决第七题和第八题，第八题可能涉及一些比较难的算法，比如DP。
    在我看来，签到题稳稳出是最重要的，你开题开得稳，自然心态就放松下来了。
  6. 保证常考的知识点你都已经掌握
  7. 可以带书进去，把一些容易忘的细节或模板记上去
  8. 期末考试是按从简单到难的，如果对自己的心态不信任，就别去看榜，容易慌。
  9. 后两三题适当计算一下时间复杂度，但是发现无论如何也优化不了的用暴力也没事。前面的题不会太为难你，尽量去思考暴力解法

知识点

以下只是常考且我想让大家考前看的知识点，有些小知识点可能有所遗漏。

暴力和贪心求法

很多问题都是只用暴力解决就好，不再赘述。但是有点题目是不能贪心写的，比如17年的D。

打图像

年年必考
说一下我解决这类的心得：

  1. 找到规律，得到上一行和下一行，左一列和右一列的关系，这是我们打图像的基础。
  2. 选择处理框架，一般有二维数组保存图像和直接输出两种办法，视情况使用
  3. 打印一部分就编译运行，确保这一步没问题再编写下一部分的代码
  4. 在程序写完后，输出最大情况和最小情况（比如n取值是0到25，分别打印一个0一个25，看看在边界情况下有没有问题）

期末考前记得从oj拿一道这类的题练练手感。

质数相关问题

这也是年年都有，难度不一。总的来说，只需要掌握埃筛就能解决绝大部分的问题了，

但是近几年也考到了前缀和和欧筛，复习时间充足还是要看看。

埃筛模板

前缀和

比如说1-10中2 3 8是符合条件的
那么我们用一个数组a[11]={0,0,1,2,2,2,2,2,3,3,3}来表示
其中 a[i]的含义是0-i中有多少个符合要求的，然后对于第x位是否符合要求只需要看a[x]-a[x-1]
（和差分互为逆运算）
前缀和常用来优化算法，降低时间复杂度，2017年的G题等都考到了前缀和与质数的结合

杨辉三角

排列组合也是比较常考的，记住杨辉三角的打表程序即可

GCD和LCM

lcm(最小公倍数)用gcd可以得知

这类题目要注意数据范围很可能爆int
因为几个很大的互质的数的最小公倍数可以很轻松的达到1e10的级别，拿不准的看到这类问题就用__int64或longlong就好

此外gcd经常应用到分数一类的问题上，将分数通分。
分数问题要注意几个特殊情况：分母为1 分母变成了0（此时无解） 分子是0。

其他细节

  1. 注意初始化！数组清空，变量赋初始值！
  2. 注意数据范围！（比如对0、1的考虑）
  3. 如果题目没说不能重复考虑重复的情况！
  4. x/a的向上取整就是(x-1)/a+1
  5. 浮点数的格式化输出都搞清楚了吗，%f %lf %g
  6. 如果对优先级没有自信，每个式子都老老实实套上括号
