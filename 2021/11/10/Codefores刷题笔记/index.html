1606A - AB Balance

传送门

题意

给定一串字符串，统计AB和BA的出现次数，通过尽量少的把s[i]变成A或变成B，使得AB与BA次数一样多。
把改变后的字符串输出

思路

一开始我是一脸懵，完全想不到思路
要去思考字符串的性质

性质1 ：连着的一串a，其实与一个a等价

简单易懂
那么aabbbabaa就可以转换成 ababa 而不改变其性质

性质2 ：bab与单个b等价

因为如果单独看bab 他能提供的ab = ba 而且他与外界接触的只有 左边右边的b，他对于外界相当于b对外界，中间的a被bb隔开，所以bab->b

那么 ababa变成aba而不改变性质

性质3 ：头尾的字符是缩串后依旧存在的

性质1和性质2的缩串只会改变中间的字符

总结

那么，一串字符串可以通过上述的缩串变成三种情况：1 aba(或bab) 2 ab 3 ba
(1 可以变成单个字符a或b)
2 3 都是不满足要求的。
由性质3，只需要判断头尾是否一样即可（或使头尾一样即可），对于任意的字符串，只需要一次操作。

1607D - Blue-Red Permutation

传送门

题意

给定一串数据
和一串BR组成的字符串
B代表对应的数据可以减小
R代表对应的数据可以增加

判断是否能搞出1-n的数据，不缺不重

思路

首先，因为没要求从前往后递增，这些数据是离散的，和位置无关。
一个很自然的想法：既然我B是用来减少的，那么我用这些数据凑前面的小的数据，R就凑大的

那么，我把B对应数据从小到大排列
把R也从小到大排列，第i个负责凑到i，这是最优的方法

code

#include<bits/stdc++.h>
using namespace std;
int const maxn = 2e5+9;
int a[maxn];
vector bj,az;
int main(){
int t;
cin>>t;
while(t–){
int n;
bj.clear();
az.clear();
cin>>n;
for(int i=0;i<n;i++){
cin>>a[i];
}
string op;
cin>>op;
for(int i=0;i<op.size();i++){
if(op[i]==’B’){
bj.push_back(a[i]);
}
else az.push_back(a[i]);
}
sort(az.begin(),az.end());
sort(bj.begin(),bj.end());
int p=1;
bool f=0;
for(int i=0;i<bj.size();i++){
if(bj[i]<p)f=1;
p++;
}
for(int i=0;i<az.size();i++){
if(az[i]>p)f=1;
p++;
}
if(f)printf(“NO\n”);
else printf(“YES\n”);
}
return 0;
}

Make Them Equal

题意

每次选一个x(1<=x<=n),把不能用整除的字符都变成c,直到全部都是c
打印需要几次和用那些数

思路

事实上就是从1到n选和其他互质的数，或者多选几次直到选到互质的数。

对于任意序列，必定能在两步之内解决：
先选n,那么1-n-1肯定都不能被整除，接着选n-1,n不被n-1整除
那么只需要考虑两步之内的方法：
0次：
显然是序列已经全是c了
1次：
把1-n所有需要变化的数分解因子，遍历一遍1-n，如果遇到没出现过的因子，那么必然无法整除其他的任何数。

Codeforces Round #757 (Div. 2)

C Divan and bitwise operations

题意

首先不得不说没仔细看题，把第二个的OR看成了XOR，导致一直没理解直到放弃。

给t个样例 n个数组成的序列，m个区间（l r x 代表从第l到第r，区间的或和为x）

求原序列的所有子序列的异或和

思路

虽然出来了，但是我的思路还是比较不清晰的：
考虑某一位i,子序列一共有2^n个，假设n个数在此位有k个数为1

那么这一位的贡献就是:

2^(i)*(C(k,1)+C(k,3)+…)*2^(n-k)=位的权值*2^(k-1)*2^(n-k)=位权*2^(n-1)

2^(n-k)是其他的可以随便放

所以先把所有的OR全部统计后*2^(n-1)

Codefores Round #760 (Div 3)

题意

选出数d 使得相邻两项不同时被d整除

思路

首先将奇偶分开讨论

能整除所有奇数位的数d必为他们的gcd,或为gcd的因子

所以遍历一遍奇偶，求得gcd1,gcd2.

此外很自然的是，d只能整除奇数或偶数位上的数据，即要去重。

所以那gcd1去判断偶数位能否被整除，能的话gcd1不行，奇数位同理。

那么为什么不需要判断gcd的因子呢？

如果gcd能整除别的位置上的数，那么gcd的因子也同样可以整除，
如果gcd不能整除，那么gcd是可行解。

所以不需要判断
