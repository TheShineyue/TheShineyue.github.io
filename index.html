1606A - AB Balance

传送门

题干

You are given a string s of length n consisting of characters a and/or b.

Let AB(s) be the number of occurrences of string ab in s as a substring. Analogically, BA(s) is the number of occurrences of ba in s as a substring.

In one step, you can choose any index i and replace si with character a or b.

What is the minimum number of steps you need to make to achieve AB(s)=BA(s)?

Reminder:

The number of occurrences of string d in s as substring is the number of indices i (1≤i≤|s|−|d|+1) such that substring sisi+1…si+|d|−1 is equal to d. For example, AB(aabbbabaa)=2 since there are two indices i: i=2 where aabbbabaa and i=6 where aabbbabaa.

Input
Each test contains multiple test cases. The first line contains the number of test cases t (1≤t≤1000). Description of the test cases follows.

The first and only line of each test case contains a single string s (1≤|s|≤100, where |s| is the length of the string s), consisting only of characters a and/or b.

Output
For each test case, print the resulting string s with AB(s)=BA(s) you’ll get making the minimum number of steps.

If there are multiple answers, print any of them.

Example
input
4
b
aabbbabaa
abbb
abbaab
output
b
aabbbabaa
bbbb
abbaaa
Note
In the first test case, both AB(s)=0 and BA(s)=0 (there are no occurrences of ab (ba) in b), so can leave s untouched.

In the second test case, AB(s)=2 and BA(s)=2, so you can leave s untouched.

In the third test case, AB(s)=1 and BA(s)=0. For example, we can change s1 to b and make both values zero.

In the fourth test case, AB(s)=2 and BA(s)=1. For example, we can change s6 to a and make both values equal to 1.

题意

给定一串字符串，统计AB和BA的出现次数，通过尽量少的把s[i]变成A或变成B，使得AB与BA次数一样多。
把改变后的字符串输出

思路

一开始我是一脸懵，完全想不到思路
要去思考字符串的性质

性质1 ：连着的一串a，其实与一个a等价

简单易懂
那么aabbbabaa就可以转换成 ababa 而不改变其性质

性质2 ：bab与单个b等价

因为如果单独看bab 他能提供的ab = ba 而且他与外界接触的只有 左边右边的b，他对于外界相当于b对外界，中间的a被bb隔开，所以bab->b

那么 ababa变成aba而不改变性质

性质3 ：头尾的字符是缩串后依旧存在的

性质1和性质2的缩串只会改变中间的字符

总结

那么，一串字符串可以通过上述的缩串变成三种情况：1 aba(或bab) 2 ab 3 ba
(1 可以变成单个字符a或b)
2 3 都是不满足要求的。
由性质3，只需要判断头尾是否一样即可（或使头尾一样即可），对于任意的字符串，只需要一次操作。

1607D - Blue-Red Permutation

传送门

题面

D. Blue-Red Permutation
time limit per test1 second
memory limit per test256 megabytes
inputstandard input
outputstandard output
You are given an array of integers a of length n. The elements of the array can be either different or the same.

Each element of the array is colored either blue or red. There are no unpainted elements in the array. One of the two operations described below can be applied to an array in a single step:

either you can select any blue element and decrease its value by 1;
or you can select any red element and increase its value by 1.
Situations in which there are no elements of some color at all are also possible. For example, if the whole array is colored blue or red, one of the operations becomes unavailable.

Determine whether it is possible to make 0 or more steps such that the resulting array is a permutation of numbers from 1 to n?

In other words, check whether there exists a sequence of steps (possibly empty) such that after applying it, the array a contains in some order all numbers from 1 to n (inclusive), each exactly once.

Input
The first line contains an integer t (1≤t≤104) — the number of input data sets in the test.

The description of each set of input data consists of three lines. The first line contains an integer n (1≤n≤2⋅105) — the length of the original array a. The second line contains n integers a1, a2, …, an (−109≤ai≤109) — the array elements themselves.

The third line has length n and consists exclusively of the letters ‘B’ and/or ‘R’: ith character is ‘B’ if ai is colored blue, and is ‘R’ if colored red.

It is guaranteed that the sum of n over all input sets does not exceed 2⋅105.

Output
Print t lines, each of which contains the answer to the corresponding test case of the input. Print YES as an answer if the corresponding array can be transformed into a permutation, and NO otherwise.

You can print the answer in any case (for example, the strings yEs, yes, Yes, and YES will be recognized as a positive answer).

Note
In the first test case of the example, the following sequence of moves can be performed:

choose i=3, element a3=5 is blue, so we decrease it, we get a=[1,2,4,2];
choose i=2, element a2=2 is red, so we increase it, we get a=[1,3,4,2];
choose i=3, element a3=4 is blue, so we decrease it, we get a=[1,3,3,2];
choose i=2, element a2=2 is red, so we increase it, we get a=[1,4,3,2].
We got that a is a permutation. Hence the answer is YES.

题意

给定一串数据
和一串BR组成的字符串
B代表对应的数据可以减小
R代表对应的数据可以增加

判断是否能搞出1-n的数据，不缺不重

思路

首先，因为没要求从前往后递增，这些数据是离散的，和位置无关。
一个很自然的想法：既然我B是用来减少的，那么我用这些数据凑前面的小的数据，R就凑大的

那么，我把B对应数据从小到大排列
把R也从小到大排列，第i个负责凑到i，这是最优的方法

code

#include<bits/stdc++.h>
using namespace std;
int const maxn = 2e5+9;
int a[maxn];
vector bj,az;
int main(){
int t;
cin>>t;
while(t–){
int n;
bj.clear();
az.clear();
cin>>n;
for(int i=0;i<n;i++){
cin>>a[i];
}
string op;
cin>>op;
for(int i=0;i<op.size();i++){
if(op[i]==’B’){
bj.push_back(a[i]);
}
else az.push_back(a[i]);
}
sort(az.begin(),az.end());
sort(bj.begin(),bj.end());
int p=1;
bool f=0;
for(int i=0;i<bj.size();i++){
if(bj[i]<p)f=1;
p++;
}
for(int i=0;i<az.size();i++){
if(az[i]>p)f=1;
p++;
}
if(f)printf(“NO\n”);
else printf(“YES\n”);
}
return 0;
}
